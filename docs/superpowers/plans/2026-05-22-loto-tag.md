## Changes from v1 (review pass)

1. **Schema bump is paired.** `internal/store/store.go: const schemaUserVersion` bumps 6→7 alongside `schema.sql`'s `PRAGMA`. v1 missed the const; would have move-asided the DB on every invocation.
2. **Implicit-ack moved into store layer.** Happens inside `ReleaseLocks`' tx, not `cmd_unlock.go`. `ReleaseResult` carries no `created_at`, so the ack can't live above the store. `--all` acks, `break` doesn't (orphans, per edge #6).
3. **Surfacing hook moved to `openRuntime`, not `RunContext`.** `RunContext` (cli.go) holds no store/agent. `whoami` and `version` never open a runtime — they're excluded. `check` is also excluded: it's the machine surface trixi's edit hook gates on, must not be polluted. T8 e2e demonstrates through `status`, not `whoami`.
4. **Render aligned to house style.** Uses existing `internal/render/cli.go` (key=value rows, glyph-led, RFC3339 UTC, `holderTag()` and `relToCwd()` reuse). No new `internal/render/tags.go` file.
5. **Orphan GC moves to `loto doctor --repair`.** Out of `Open()` hot path — `check` runs on every Go save via trixi's PreToolUse hook; running a `DELETE` there is wasteful when read-time filtering already hides orphans.
6. **Cap enforced transactionally inside `InsertTag`.** Drops `CountAliveForLock` from the public API (TOCTOU bait). Also drops unused `ListAliveForLock`. Adds `newTagID()` mirroring `newEventID()`.
7. **`loto tag list` deferred to v2.** Holder sees external tag IDs in the surfaced block → `loto ack <id>` works directly. Sender can inspect via `loto status <file>` (which includes their own tags per visibility matrix). Cuts the `--mine`/`--for-me`/`--all` mode question entirely.

---

**Goal:** Add a path-scoped, lock-parasitic annotation primitive (`loto tag`) so an agent who finds a file locked by a peer can leave a question/note that the lock holder is forced to see on their next loto operation. No standalone inbox; no `--to <handle>` addressing; visibility is derived from current lock state.

**Architecture:** Tags are rows in `loto.db` keyed by the host lock's identity triple `(target_canonical, owner_uuid, lock_created_at)`. When the host lock disappears (release / break / expire / takeover), tags die with it (read-time filter; lazy GC in doctor). Surfacing happens at three sites: (a) `openRuntime`-bearing commands print the holder's pending tags after primary output, (b) `lock` conflict output appends current tags via existing `EmitConflict`, (c) `status <file>` appends current tags. No new daemons, no new files, no new IPC.

**Tech Stack:** Go, SQLite (existing `loto.db`, schema bump v6→v7 paired with `const` bump), existing `internal/store` + `internal/cli` + `internal/render` packages, existing render conventions (`.claude/rules/design.md`, render/cli.go).

## Formal model (claudish)

```
Tag = {id, target, lock_owner, lock_created_at, tagger, text, created_at, acked_at?}

Host(tag) = locks.row where (target_canonical, owner_uuid, created_at)
                       == (tag.target, tag.lock_owner, tag.lock_created_at)

Alive(tag)    ≡ Host(tag) ∃ ∧ tag.acked_at = ∅
Orphaned(tag) ≡ Host(tag) = ∅          → filtered on read, GC'd lazily by doctor --repair
Acked(tag)    ≡ tag.acked_at ≠ ∅       → filtered on read

Recipient(tag) = lock_owner  (∵ Host(tag).owner_uuid)
Self-tag(tag)  ≡ tag.tagger = tag.lock_owner

EchoToHolder(agent) = { t : Alive(t) ∧ Recipient(t) = agent ∧ ¬Self-tag(t) }
EchoOnConflict(file, agent) = { t : Alive(t) ∧ t.target = canon(file) ∧ t.tagger ≠ agent }
EchoOnStatus(file) = { t : Alive(t) ∧ t.target = canon(file) }

Cap(file, lock_instance) = count({Alive(t) ∧ t.target = file
                                  ∧ (t.lock_owner, t.lock_created_at) = lock_instance})
                         ≤ 5    -- enforced inside InsertTag tx
```

**Lifecycle:**

| event                               | effect on tags on host lock                            |
|-------------------------------------|--------------------------------------------------------|
| `loto unlock <file>` by holder      | `AckAllForLock` inside `ReleaseLocks` tx               |
| `loto unlock --all` by holder       | acks tags on each released lock (same store path)      |
| `loto unlock --force` (break) by 3rd| tags orphaned (host row gone), no ack written          |
| lock expiry                         | tags orphaned                                          |
| holder re-acquires after release    | previous tags stay dead (new `lock_created_at`)        |
| `loto ack <id>` by holder           | one tag acked                                          |
| `loto doctor --repair`              | hard-deletes orphaned + long-acked tag rows            |

**Invariants:**

```
I1: tag.tagger ≠ tag.lock_owner → external; counted for cap, in EchoToHolder
I2: tag.tagger = tag.lock_owner → self-annotation; counted for cap, NOT in EchoToHolder
I3: Alive(t) ⟹ Host(t) ∃              -- parasitic on lock
I4: acked_at monotonic (set once, never unset)
I5: text append-only (no UPDATE)
I6: cap enforced inside InsertTag's tx (no TOCTOU)
```

## Visibility matrix

| caller \ tag                | external (others wrote it) | self-tag (caller wrote it) |
|-----------------------------|----------------------------|-----------------------------|
| holder runs runtime cmd     | ✓ trailing block           | ✗ (no self-spam)            |
| holder runs `ack`           | ✓ (acks it)                | n/a                         |
| holder runs `unlock`        | ✓ (implicit ack)           | ✓ (implicit ack)            |
| non-holder `status f`       | ✓                          | ✓                           |
| non-holder `lock f` (fail)  | ✓ via EmitConflict         | ✓ via EmitConflict          |

‡ `loto tag list` is deferred (v2). Sender who wants to inspect their own pending tags uses `loto status <file>`.

‡ "Runtime cmd" = any command that calls `openRuntime`: `lock`, `unlock`, `check`, `status`, `doctor`. **`check` is explicitly excluded from surfacing** — see §Hook surface.

## Edge cases

| # | scenario                                                            | resolution                                                                 |
|---|---------------------------------------------------------------------|----------------------------------------------------------------------------|
| 1 | tag a file with no lock                                             | reject: `✗ <file> not locked — acquire it yourself`                        |
| 2 | tag a file caller already locks                                     | accept (self-annotation), no self-echo                                     |
| 3 | tag where caller is non-holder                                      | accept (the normal case)                                                   |
| 4 | 6th tag attempt on same (file, lock)                                | reject: `✗ tag cap reached on <file> (5) — escalate channel`               |
| 5 | tag, then holder releases before reading                            | tag orphaned silently; sender not notified (v1 non-goal)                   |
| 6 | tag, then holder loses lock to `break --force`                      | tags orphaned, no implicit ack written                                     |
| 7 | tag, holder releases, then re-acquires                              | new `lock_created_at` → previous tag stays dead                            |
| 8 | tag rows survive store reopens / crashes                            | durable rows like locks/events                                             |
| 9 | concurrent `tag` calls at cap boundary                              | cap check inside same tx as INSERT → second tx errors `ErrTagCapReached`   |
| 10| `ack` an already-acked tag                                          | no-op, exit 0 (idempotent)                                                 |
| 11| `ack` an orphaned tag                                               | no-op, exit 0                                                              |
| 12| `ack` someone else's tag                                            | reject: `✗ not addressed to you`                                           |
| 13| `loto tag` while DB busy                                            | inherits store flock + busy retry                                          |

## Hook surface (re-grounded)

Existing dispatch (`internal/cli/cli.go::RunContext`) is a thin registry lookup with no `*runtime`. Surfacing **cannot** live in dispatch. The honest hook is `runtime.go::openRuntime` — every command that needs the store/agent calls it.

```
openRuntime callers (verified)        surfaces tags?
─────────────────────────────────────────────────────
cmd_lock.go        (lock)             ✓ trailing block; conflict block via EmitConflict
cmd_unlock.go      (unlock)           ✓ trailing block
cmd_status.go      (status)           ✓ per-file block inline + trailing block
cmd_doctor.go      (doctor)           ✓ trailing block
cmd_check.go       (check)            ✗ MACHINE SURFACE — excluded
─────────────────────────────────────────────────────
cmd_whoami.go      (whoami)           ✗ never opens runtime
cmd_version.go     (version)          ✗ never opens runtime
```

Mechanism: a helper `(r *runtime) DeferredTagFooter(w io.Writer)` registered via `defer` in each opted-in command, *after* primary output. Implementation: `store.ListAliveForHolder(ctx, r.Agent.UUID)` + `render.EmitTagFooter`. `check` opts out by not calling the deferred helper.

‡ `check` exclusion rationale: trixi's `loto-guard.sh` PreToolUse hook parses `loto check` output on every `.go` save. Prepending/appending variable tag content breaks `output_glyphs_test.go` and `help_golden_test.go`. Holders see their tags via the next `lock`/`unlock`/`status`/`doctor` instead.


```go
package store

import (
    "context"
    "database/sql"
    "errors"
    "time"
)

var ErrTagCapReached = errors.New("loto: tag cap reached")
var ErrTagNotMine    = errors.New("loto: tag not addressed to caller")

const tagCap = 5

type NewTag struct {
    Target, LockOwner, Tagger, Text string
    LockCreatedAt                   int64
}

type Tag struct {
    ID, Target, LockOwner, Tagger, Text string
    LockCreatedAt, CreatedAt            int64
    AckedAt                             *int64
}

func newTagID() string { return "t-" + randHex(8) }  // mirrors newEventID idiom

func (s *Store) InsertTag(ctx context.Context, t NewTag) (string, error) {
    tx, err := s.beginTx(ctx)
    if err != nil { return "", err }
    defer tx.Rollback()

    // host-lock existence + cap, inside the same tx
    var n int
    err = tx.QueryRowContext(ctx, `
      SELECT COUNT(*) FROM tags
      WHERE target_canonical=? AND lock_owner_uuid=? AND lock_created_at=?
        AND acked_at IS NULL
        AND EXISTS (SELECT 1 FROM locks l
                    WHERE l.target_canonical=? AND l.owner_uuid=? AND l.created_at=?)`,
      t.Target, t.LockOwner, t.LockCreatedAt,
      t.Target, t.LockOwner, t.LockCreatedAt).Scan(&n)
    if err != nil { return "", err }
    if n >= tagCap { return "", ErrTagCapReached }

    // also verify host lock exists (covers edge #1 from caller)
    var ok int
    err = tx.QueryRowContext(ctx, `
      SELECT 1 FROM locks WHERE target_canonical=? AND owner_uuid=? AND created_at=?`,
      t.Target, t.LockOwner, t.LockCreatedAt).Scan(&ok)
    if errors.Is(err, sql.ErrNoRows) { return "", ErrNoHostLock }
    if err != nil { return "", err }

    id := newTagID()
    now := time.Now().UnixMilli()
    _, err = tx.ExecContext(ctx, `
      INSERT INTO tags(id,target_canonical,lock_owner_uuid,lock_created_at,
                       tagger_uuid,text,created_at)
      VALUES(?,?,?,?,?,?,?)`,
      id, t.Target, t.LockOwner, t.LockCreatedAt, t.Tagger, t.Text, now)
    if err != nil { return "", err }
    return id, tx.Commit()
}

func (s *Store) ListAliveForHolder(ctx context.Context, ownerUUID string) ([]Tag, error)
func (s *Store) ListAliveForTarget(ctx context.Context, target string) ([]Tag, error)
func (s *Store) Ack(ctx context.Context, tagID, byUUID string) error
func (s *Store) AckAllForLock(ctx context.Context, target, owner string, createdAt int64) error
```

`ListAliveForHolder` JOIN'd SQL (deterministic order, self-tag suppression):

```sql
SELECT t.* FROM tags t
JOIN locks l
  ON l.target_canonical = t.target_canonical
 AND l.owner_uuid       = t.lock_owner_uuid
 AND l.created_at       = t.lock_created_at
WHERE l.owner_uuid  = ?      -- :ownerUUID
  AND t.tagger_uuid <> ?     -- :ownerUUID (no self-echo)
  AND t.acked_at IS NULL
ORDER BY t.created_at ASC, t.id ASC;
```

`ListAliveForTarget` same minus the self-tag filter (everyone sees everyone's tags on conflict/status).

## Non-goals (v1)

- **`loto tag list`** — deferred. Holder sees IDs in surfaced block; sender inspects via `loto status <file>`.
- **Sender notification on orphaned tag.** No "your tag went stale" notice; non-goal per discussion.
- **`loto tag rm <id>` (sender-side delete pre-ack).** Workaround: wait for orphan, or hold-then-tag-then-unlock self-ack.
- **Editing tag text.** I5 append-only. Mistake → ack + new tag.
- **Cross-repo tags.** Per-repo `loto.db`. Out of scope.
- **Broadcast / multi-recipient.** Current-holder rule only.
- **Tag metadata (priority, type).** Just text.
- **TUI/dashboard.** Render conventions apply; no separate human surface.
