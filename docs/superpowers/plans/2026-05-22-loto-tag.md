# loto tag — inter-agent annotations bound to locks

> **For agentic workers:** Use dk:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Bead:** loto-eyk
>
> **Status:** draft, pre-review. Names and signatures stable enough to review; cosmetic details (exact glyphs, exact error wording) may shift in implementation.

**Goal:** Add a path-scoped, lock-parasitic annotation primitive (`loto tag`) so an agent who finds a file locked by a peer can leave a question/note that the lock holder is forced to see on their next loto operation. No standalone inbox; no `--to <handle>` addressing; visibility is derived from current lock state.

**Architecture:** Tags are rows in `loto.db` keyed by the host lock's identity triple `(target_canonical, owner_uuid, lock_created_at)`. When the host lock disappears (release / break / expire / takeover), tags die with it. Surfacing happens at three sites: (a) every loto command prints the holder's pending tags at top of output, (b) `try`/`lock` conflict output appends current tags, (c) `status <file>` appends current tags. No new daemons, no new files, no new IPC. Reuses existing store/runtime patterns.

**Tech Stack:** Go, SQLite (existing `loto.db`, schema bump v6→v7), existing `internal/store` + `internal/cli` packages, existing render conventions (`.claude/rules/design.md`, nug `32f0ece29b72`).

## Formal model (claudish)

```
Tag = {id, target, lock_owner, lock_created_at, tagger, text, created_at, acked_at?}

Host(tag) = locks.row where (target_canonical, owner_uuid, created_at)
                       == (tag.target, tag.lock_owner, tag.lock_created_at)

Alive(tag)    ≡ Host(tag) ∃ ∧ tag.acked_at = ∅
Orphaned(tag) ≡ Host(tag) = ∅          → filtered on read, GC'd opportunistically
Acked(tag)    ≡ tag.acked_at ≠ ∅       → filtered on read

Recipient(tag) = lock_owner  (∵ Host(tag).owner_uuid)
Self-tag(tag)  ≡ tag.tagger = tag.lock_owner

EchoToHolder(agent) = { t : Alive(t) ∧ Recipient(t) = agent ∧ ¬Self-tag(t) }
EchoOnConflict(file, agent) = { t : Alive(t) ∧ t.target = canon(file) ∧ t.tagger ≠ agent }
EchoOnStatus(file) = { t : Alive(t) ∧ t.target = canon(file) }

Cap(file, lock_instance) = count({Alive(t) ∧ t.target = file
                                  ∧ (t.lock_owner, t.lock_created_at) = lock_instance})
                         ≤ 5
```

**Lifecycle events:**

| event                           | effect on tags on host lock                      |
|---------------------------------|--------------------------------------------------|
| `loto unlock <file>` by holder  | implicit ack of all tags on that lock            |
| `loto break <file>` by 3rd      | tags orphaned (lock_owner row gone) → invisible  |
| lock expiry (TTL)               | tags orphaned → invisible                        |
| holder re-acquires after release| previous tags stay dead (new `lock_created_at`)  |
| `loto ack <id>` by holder       | one tag acked                                    |
| tagger deletes own (`tag rm`)   | one tag removed (out of scope for v1 — see §Non-goals) |

**Invariants:**

```
I1: tag.tagger ≠ tag.lock_owner → tag is "external" (counted for cap, EchoToHolder)
I2: tag.tagger = tag.lock_owner → tag is "self-annotation"
                                  (counted for cap, NOT EchoToHolder, still in EchoOnConflict/Status)
I3: Alive(t) ⟹ Host(t) ∃                          # parasitic on lock
I4: ack is monotonic — acked_at, once set, never unset
I5: no UPDATE of text — tags are append-only; mistake → ack + new tag
```

## Visibility matrix

| caller \ tag                | external tag (others wrote it) | self-tag (caller wrote it) |
|-----------------------------|--------------------------------|-----------------------------|
| holder runs any `loto …`    | ✓ top-of-output                | ✗ (no self-spam)            |
| holder runs `loto tag list` | ✓                              | ✓ (sender browses own)      |
| holder runs `loto unlock`   | ✓ (acks them)                  | ✓ (acks them)               |
| non-holder `loto status f`  | ✓                              | ✓                           |
| non-holder `try`/`lock` fail| ✓                              | ✓                           |
| non-holder `loto tag list`  | own tags only (any file)       | n/a                         |

`tag list` filtering rule: `--mine` (default for non-holder) shows tags the caller authored; `--for-me` (default for holder) shows tags addressed to caller-as-holder. `--all` overrides both.

## Edge cases — explicit

| # | scenario                                                            | resolution                                                                 |
|---|---------------------------------------------------------------------|----------------------------------------------------------------------------|
| 1 | tag a file with no lock                                             | reject: `✗ <file> not locked — acquire it yourself`                        |
| 2 | tag a file caller already locks                                     | accept (self-annotation), no self-echo                                     |
| 3 | tag where caller is not holder and not lock-querying                | accept (the normal case)                                                   |
| 4 | 6th tag attempt on same (file, lock)                                | reject: `✗ tag cap reached on <file> (5) — escalate channel`               |
| 5 | tag, then holder releases before reading                            | tag orphaned silently; sender not notified (per §Non-goals)                |
| 6 | tag, then holder loses lock to `break --force`                      | same as #5: orphaned, no notification                                      |
| 7 | tag, holder releases, then re-acquires                              | new `lock_created_at` → previous tag stays dead                            |
| 8 | tag survives store reopens / process crashes                        | tag rows are durable in `loto.db` like locks                               |
| 9 | concurrent `tag` calls at the cap boundary                          | INSERT inside a transaction that re-checks cap; second tx errors out      |
| 10| `ack` an already-acked tag                                          | no-op, exit 0 (idempotent)                                                 |
| 11| `ack` an orphaned tag                                               | no-op, exit 0 (already invisible)                                          |
| 12| `ack` someone else's tag (caller is not holder)                     | reject: `✗ not addressed to you`                                           |
| 13| `loto tag` while DB is locked by another process                    | inherit existing flock+busy retry behavior from store                      |

## Schema delta

```sql
-- schema.sql (additions); PRAGMA user_version bumped 6 → 7
CREATE TABLE IF NOT EXISTS tags (
  id                TEXT PRIMARY KEY,
  target_canonical  TEXT NOT NULL,
  lock_owner_uuid   TEXT NOT NULL,
  lock_created_at   INTEGER NOT NULL,
  tagger_uuid       TEXT NOT NULL,
  text              TEXT NOT NULL,
  created_at        INTEGER NOT NULL,
  acked_at          INTEGER  -- NULL = pending
);
CREATE INDEX IF NOT EXISTS idx_tags_host
  ON tags(target_canonical, lock_owner_uuid, lock_created_at);
CREATE INDEX IF NOT EXISTS idx_tags_holder_pending
  ON tags(lock_owner_uuid, acked_at);
```

‡ No FK to `locks` (locks row deletes are routine, FK would force cascades and complicate `break`/expiry paths). Orphan filter at read time + opportunistic GC on store-open keeps it lazy. Mirrors existing pattern (events table also has no FK to locks).

‡ `PRAGMA user_version = 7` — existing `MoveCorruptAside` on mismatch behavior applies. Anyone with a v6 DB gets a fresh store. Acceptable for this project's "claim → re-acquire" model. Document in NORTH_STAR alongside the change.

## Non-goals (v1) — call out for reviewer

- **Sender notification when tag dies orphaned.** Per discussion edge #1 above: tagger gets no "your tag went stale" notice. If review pushes back, this is a small addition: a `notifications` table or a per-agent `seen_orphans` cursor — but it doubles the lifecycle complexity. Defer to a follow-up bead.
- **`loto tag rm <id>` (sender deletes own pre-ack).** Workaround: tagger can wait for orphan, or hold-then-tag-then-unlock to self-ack. If common, add later.
- **Editing tag text.** Invariant I5. Mistake → ack + new tag.
- **Cross-repo tags.** Tags live in per-repo `loto.db`. Outside scope of the "siblings in one repo" use case.
- **Tag broadcast / multi-recipient.** No `--to *`. The current-holder rule is the only addressing.
- **Tag metadata (priority, type, category).** Just text. If structure is wanted, do it in the text body (sender's discipline).
- **A dashboard / TUI for tags.** `loto status` is the human-readable surface and is the only place dashboard rules apply.

## Open questions for review

1. **Storage of `lock_created_at` redundancy.** Could collapse to a `lock_id` FK if we add a stable PK to `locks`. Currently `locks.target_canonical` is the PK and `created_at` is mutable-free in practice but not declared so. Worth promoting `lock_id` (uuid) to a real column? Trade-off: bigger schema change, but simpler joins. → I'd defer to a separate bead; current triple works.
2. **Glyph for the tag block.** Design rules list ✓ ∇ ‡ † ∅ φ → ◯ ≈ ∞. Nothing maps obviously to "annotation." Candidates: `‡` (already "critical"), a new bareword `tags:N`, or a φ-prefixed file ref. Plan currently shows `tags:N` — happy to change pre-impl.
3. **Where exactly to hook `surfaceHolderTags`.** The current `runtime.go` doesn't have a single output-funnel; each `cmd_*.go` writes directly. Cleanest is a small wrapper in `main`/dispatch that prints the block before invoking the command's `Run`. Alternative: an early line in each `cmd_*.go`. Wrapper is DRYer; I'd go wrapper. Confirm preference.
4. **Cap value (5).** Soft-coded constant. Make it configurable via env? I'd say no for v1 — if it bites, raise the constant.
5. **`loto tag list` defaults.** Plan says: defaults to `--for-me` if caller holds any locks, else `--mine`. Could also always require an explicit flag. Defaulting feels more ergonomic but is mode-y.
